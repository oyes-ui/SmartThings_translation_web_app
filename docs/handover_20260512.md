# Handover & System Update Report (2026-05-12)

## 1. 프롬프트 아키텍처 V2 주요 변경 사항

`prompt_modules.py`와 `prompt_builder.py`의 구조를 개편하여 컨텍스트 기반의 정교한 로컬라이제이션 규칙을 적용했습니다.

### 1.1 용어집(Glossary) 판정 로직 고도화
- **기존 방식**: `GLOBAL_GLOSSARY_STANDARDS`를 통한 일괄 적용.
- **신규 방식**: `GLOSSARY_CONTEXT_RULES`를 도입하여 `row_key`에 따라 모드를 분기합니다.
  - **`title_button` 모드**: `row_key`에 `title`, `button`이 포함되거나 숫자로 끝나는 경우. 용어집 단어에 대괄호를 붙이지 않습니다.
  - **`description_disclaimer` 모드**: `description`, `disclaimer`가 포함된 경우. 언어별 지정 대괄호(`[]` 또는 Japanese `「」`)를 사용합니다.
- **단일 진실 공급원**: `PromptBuilder.get_glossary_context_mode(row_key)`가 이 모든 판정의 기준이 됩니다.

### 1.2 언어 모듈 분리 및 최적화
- **포르투갈어 분리**: `Brazilian Portuguese`와 `European Portuguese`로 규칙을 분리하여 시장별 어휘/문법 혼용을 방지했습니다.
- **중국어 분리**: `Simplified Chinese`(Mainland)와 `Traditional Chinese`(Taiwan)로 분리하여 용어 체계 정합성을 높였습니다.
- **테스트 완료**: `tests/045_TEST.xlsx`와 `glossary_TEST_ver2.csv`를 통해 모든 언어의 매핑과 규칙 로드(PASS)를 확인했습니다.

## 2. 프롬프트 부하(Heaviness) 분석 결과

- **Standard Mode**: 약 **2,000자 / 300단어**. (안정적)
- **Worst-Case (RAG 포함)**: 셀당 약 **3,000 ~ 3,500 토큰** 예상.
- **결론**: 현재의 모듈형 구조는 모델의 컨텍스트 한계 내에 있으나, 대량 병렬 처리 시 누적 토큰 비용과 지연 시간(Latency) 관리가 핵심 과제입니다.

## 3. 디버깅 및 개발 도구

- **Prompt Inspector (`static/demo.html`)**: 새로 추가된 디버그 페이지입니다. `row_key`와 언어 설정을 바꿔가며 실시간 프롬프트 조립 상태를 육안으로 확인할 수 있습니다.
- **핵심 API**:
  - `/api/prompt_modules`: 모듈별 활성화 상태 및 `Context mode` 확인.
  - `/api/preview_prompt_blocks`: 최종 조립되는 프롬프트의 섹션별 데이터 반환.

## 4. 향후 작업 과제 (Backlog)

- **RAG Trimming**: 유사 사례가 너무 길 경우 토큰 절감을 위해 텍스트를 요약하거나 개수를 제한하는 로직 추가 검토.
- **Concurrency Monitoring**: 대량 시트 번역 시 API Rate Limit에 걸리지 않도록 동시 처리 수(`max_concurrency`) 최적화.
- **BX 가이드라인 정리**: 현재 비활성화 상태인 BX Style 규칙들의 유지 여부 또는 경량화 결정.
