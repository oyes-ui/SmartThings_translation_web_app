# 삼성제공 vs DF보유 파일명·지시문서 기반 범위 확인

## 기준

| 구분 | 기준 | 비고 |
| --- | --- | --- |
| DF보유 | `[DF보유]/1_Global`, `[DF보유]/2_Local` | 파일명에서 Story 번호, 언어 코드, 날짜 suffix 추출 |
| 삼성제공 | `[삼성제공]` 하위 글로벌향/중국향/기타 언어 파일 | DRM 파일은 내용 접근 불가이므로 파일명·폴더 기준 |
| DF 전달용 지시 문서 | `(CX Center) Explore_2.0_Story_CHN_1016_f_DF 전달용.xlsx` | 시트명, E열 리뷰 텍스트, 셀 fill 하이라이트 후보 추출 |

> 삼성제공 DRM 파일은 첫 바이트가 `<## NASCA DRM FILE - VER1.00 ##>`로 시작한다. 따라서 일반 `.xlsx` ZIP 구조가 아니며 Python/openpyxl로 내용 접근이 불가하다.

> 표 안의 <span style="background-color:#ffd6d6;color:#9b0000;font-weight:600">빨간색 셀</span>은 보유 구성·문구·section 구조가 달라서 확인이 필요한 항목이다. `//update` 같은 관리 메타 차이는 빨간색 대상에서 제외했다.

> 표 안의 <span style="background-color:#fff2cc;color:#7a4d00;font-weight:600">노란색 셀</span>은 중국 미제공 또는 의도적 삭제 범위로 보이는 항목이다. 누락 이슈라기보다 삭제/제외가 맞는지 확인할 항목이다.

## 수량 정산 및 현재 해석

삼성은 글로벌 28개 + 지역향 20개(중국향 16개, 기타 언어 4개)로 총 48개다. DF는 Global 31개 + Local 23개로 총 54개다. 아래는 파일 수가 6개 차이 나는 이유를 구성 기준으로 정산한 것이다.

| 정산 항목 | DF 대비 증감 | 대상 | 현재 해석 |
| --- | --- | --- | --- |
| DF에만 있는 글로벌 | +2 | 025, 039 | 현행 운영 범위 차이 여부 확인. |
| DF CN은 있으나 삼성 CN 없음 | +5 | <span style="background-color:#ffd6d6;color:#9b0000;font-weight:600">001, 004, 005, 006, 038</span> | <span style="background-color:#ffd6d6;color:#9b0000;font-weight:600">중국향 제공 제외/누락 사유 확인 필요</span> |
| 삼성에만 있는 지역향 | -1 | 006_IN | 삼성 최신 4-section 구성. DF Global과 순서 차이 및 문구 1건 소폭 변경. |
| Global 폴더의 한국향 대응 | 0 | DF Global 051 ↔ 삼성 051_KR | 051은 한국향 스토리. DF 폴더 위치와 무관하게 삼성 KR 파일이 대응하며 본문은 동일, //update만 다름. |
| 합계 | +6 | DF 54개 - 삼성 48개 | 5건 CN 미제공 사유가 확인되면 개수 차이는 구성상 설명 가능. |

| 006_IN | DF Global 006 / DF CN 006 | <span style="background-color:#ffd6d6;color:#9b0000;font-weight:600">가능</span> | <span style="background-color:#ffd6d6;color:#9b0000;font-weight:600">삼성 IN은 최신 4-section 구성. DF Global과 section 순서는 다르며, 수면 설명에 ‘숙면을 방해하는 에어컨’ 문구가 1건 추가됨. 삼성 CN 파일은 없어 CN 최신 반영 여부 확인 필요.</span> |

## 셀 내용 대조 기준

| 항목 | 기준 |
| --- | --- |
| 비교 대상 | DF보유 `1_Global`과 `2_Local`의 같은 Story 파일 |
| 비교 시트 | 각 워크북의 첫 번째 시트 |
| 비교 키 | `B열 key` |
| 비교 값 | `C열 text` |
| 값 변경 | 같은 key가 양쪽에 있으나 C열 값이 다름 |
| Global only | Global에는 있으나 Local에는 없는 key |
| Local only | Local에는 있으나 Global에는 없는 key |
| 메타 변경 | `//update`, `//language`, 관리 메모 등 콘텐츠가 아닌 관리성 차이 |
| 구조 차이 | story/section key가 한쪽에만 있어 실제 번역 범위가 달라지는 차이 |
